## 三、授权码状态流转

```
active（激活）  ←→  disabled（已禁用）
      ↓                ↓
   revoked（已撤销，不可逆）
      ↓
   expired（已过期）
```

- **active → disabled**：管理员手动禁用（临时停用，可恢复）
- **disabled → active**：管理员手动启用（恢复使用）
- **active/disabled → revoked**：管理员撤销（永久失效，不可逆）
- **active → expired**：超过有效期，**需定时任务触发更新**，非实时自动变更**

## 五、离线激活（请求/响应文件机制）

适用于客户端完全无法连接外网的环境。

**流程**：

```
[离线客户端]
  1. 生成离线激活请求（license_key + device_fingerprint）
      获得 request.json
  2. 将 request.json 手动传输到联网机器

[联网机器]
  3. 提交请求文件，服务端生成响应文件（RSA 签名）
      下载 response.json
  4. 将 response.json 手动传回离线客户端

[离线客户端]
  5. 验证响应文件（RSA 签名 + 有效期）
      验证通过 → 激活成功
```

| # | 功能 | 说明 |
|---|------|------|
| 5.1 | 生成离线激活请求 | `POST /api/v1/offline/request/gen` |
| 5.2 | 下载离线请求文件 | `GET /api/v1/offline/request/:token/download` |
| 5.3 | 处理离线激活 | `POST /api/v1/offline/activate/:token`，生成 RSA 签名响应文件 |
| 5.4 | 验证离线响应文件 | `POST /api/v1/offline/verify`，客户端验证签名和有效期 |

## 六、离线授权文件验证（方案1：无宽限期）

适用于客户端无法联网，但仍需验证授权有效性的场景。

**原理**：
1. 客户端联网时，下载**离线授权文件**（包含授权信息 + RSA 签名）
2. 客户端将文件保存到本地
3. 离线时，客户端验证本地授权文件：
   - 验证 RSA 签名（使用服务端公钥）
   - 检查授权有效期（`valid_from` / `valid_to`）
4. **无需定期联网**，文件在授权有效期内始终有效

> ⚠️ **时钟篡改风险**：此方案只校验有效期，用户可通过修改系统时钟绕过过期检查。

| # | 功能 | 说明 |
|---|------|------|
| 6.1 | 下载离线授权文件 | `GET /api/v1/license/:licenseKey/offline-auth-file`，需 JWT 认证 |
| 6.2 | 验证本地授权文件 | 客户端使用 SDK 验证本地授权文件的签名和有效期 |
| 6.3 | 授权文件内容 | 包含 license_key、company、product_keys、valid_from、valid_to、activated_count、max_activations、signature、certificate |
| 6.4 | 离线授权文件构建 | 服务端使用 RSA 私钥签名，构建完整的离线授权文件 |
| 6.5 | 离线授权文件验证 | 客户端使用 RSA 公钥验证签名和有效期 |

## 十四、数据与配置

### 14.1 数据库表结构

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `users` | 用户账号 | 用户名（唯一）、密码哈希、邮箱（唯一索引） |
| `licenses` | 授权码 | license_key（唯一）、product_keys（JSON）、status、valid_from、valid_to、max_activations、activated_count、notified_7days、notified_1day、notified_expired、is_permanent、hardware_ids（JSON） |
| `activations` | 激活记录 | license_id、device_fingerprint、method（online/offline）、activated_at、last_verified_at、client_version、client_ip、is_active |
| `offline_requests` | 离线激活请求记录 | license_id、request_token、device_fingerprint、response_file、status、expires_at |
| `products` | 产品列表 | product_key（唯一）、name、description |
| `applications` | 授权码申请记录 | applicant_name、email、phone、product_keys（JSON）、duration_months、max_activations、description、status（pending/approved/rejected）、admin_note、license_key |
| `operation_logs` | 操作日志 | admin_id、admin_name、action、target、detail、ip、created_at |
